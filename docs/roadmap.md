# 路线图

路线图会随着实际使用反馈调整，以下是按优先级粗略整理的方向。

## 近期

- Field Omission 的 UI 配置入口（目前 core 已支持，UI 尚未开放）
- 更完善的规则匹配能力：query/body/header 维度（在不破坏现有行为的前提下逐步扩展）
- API 解析增强：支持更多组织方式（例如多文件、导入 URL 常量、简单拼接等）

## 中期

- 基于 `Response` TS 类型的 mock 模板生成：
  - 从 TS 类型构建 schema（或基于 JSDoc/示例）生成初始 `result`
  - UI 一键生成/刷新，并允许手工编辑
  - 需要在构建期做类型解析（`tsconfig`/类型展开/跨文件引用）
- 规则导入/导出（UI + 文件）
- 批量编辑与批量应用（多选 API）

## 长期

- 更丰富的网络模拟：带宽/分片、重试抖动、阶段性错误等
- 可插拔的 mock 数据生成器（faker/seed）
- 团队共享规则（远程存储/同步）
